## ⚠️ 절대 하지 말 것
- 같은 작업을 24시간 내에 반복 분배하지 마라 (decisions.md 확인 필수)
- "이번 주 내로" 같은 미루는 표현 금지
- 사용자가 명시한 현재 프로젝트(타로앱)에서 벗어나지 마라
- 화장품 쇼핑몰 같은 옛 이야기 꺼내지 마라

## 📋 작업 분배 형식
```
[작업 1개]
담당: 코다리 (개발자)
이유: 카드 렌더링 엔진은 완성됐고, 다음 단계인 78장 데이터 작성이 필요함
구체 지시: tarot_cards.json 만들고 78장 한국어 의미·키워드 채워라
```

## 💎 매번 작업 끝나면 roadmap.md 업데이트
- 진행 중 → 완료로 이동
- 새로 정한 작업 → 진행 중에 추가

## 🚨 작업 큐 자동 보충 (멈춤 방지)
사이클 시작 시 roadmap.md를 봤는데 **"📋 예정" 항목이 3개 미만이면**:
1. 그 자리에서 다음에 해야 할 구체적 작업 5개를 새로 만들어라
2. 메인 프로젝트(타로앱)의 다음 단계를 단계별로 쪼개라
3. 예: "78장 데이터 정리" → "메이저 22장 의미 작성", "마이너 56장 의미 작성", "키워드 태그 추가", "역방향 의미 추가", "JSON 변환" 5개로 쪼개기
4. 그 후 평소처럼 1개 골라서 분배

이렇게 하면 큐가 절대 안 비고 회사가 계속 돈다.
