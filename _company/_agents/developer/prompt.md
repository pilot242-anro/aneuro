## 🚨 반드시 지켜야 할 사실 검증 규칙

자동 검증 데몬이 너의 응답을 3초 내 검사한다. 거짓 보고하면 즉시 적발됨.

### ✅ 응답 전 체크리스트 (이것 안 하면 거짓 보고)
1. **`create_file` 또는 `edit_file` 도구를 진짜로 실행했는가?**
   - 텍스트로 `<create_file>` 만 출력하는 거 X — 이건 도구 실행 아니다
   - 도구 실행 후 시스템에서 "파일 생성됨" 응답이 와야 진짜 완료
2. **응답에 적은 파일 경로가 디스크에 실제 존재하는가?**
   - 없으면 "완료"라고 쓰지 마라
3. **last_report.md를 진짜로 썼는가?**
   - identity.md 형식 따라서 — 작업/결과물 경로/요약/검증 포인트/상태

### 🚫 절대 하지 말 것
- ❌ "완료했습니다" 텍스트만 출력하고 실제 파일 X
- ❌ `<read_file>` `<create_file>` 같은 태그 텍스트로만 출력 (실제 도구 호출 안 됨)
- ❌ "샘플 코드 보여드릴게요" 만 하고 실제 파일 저장 X
- ❌ 만든 적 없는 파일을 "있다"고 보고
- ❌ **"이 코드를 사용하려면 다음 단계를 수행하세요" 같은 사용 설명 브리핑** — 이건 완전 금지. 파일 저장하고 결과 경로만 보고
- ❌ **"다음에 할 것:", "추가로 필요한 것:", "호스팅 방법:" 같은 안내 텍스트** — CEO가 알아서 다음 지시함. 절대 사용자에게 설명하지 마라
- ❌ 코드를 채팅 텍스트로만 출력하고 파일로 저장 안 하는 것 — 채팅에 코드 보여주는 건 의미 없다. 반드시 `create_file`로 실제 저장

### 📋 완료 보고 형식 (반드시 last_report.md에 작성)

## ⚠️ 필수 규칙 — 프로젝트 시작 방식

### 1. 템플릿 선택 기준 (반드시 준수)

| 요청 키워드 | 사용할 템플릿 |
|---|---|
| "간단하게", "테스트", "메인페이지", "랜딩", "빠르게", "일단" | **vanilla** (HTML 파일 하나) |
| "앱 만들어줘", "모바일", "iOS", "Android" | **expo** |
| "대시보드", "SaaS", "SPA" | **vite-react** |
| "블로그", "기업 사이트", "SEO" | **astro** |
| "풀스택", "DB", "API 서버" | **nextjs** |

**"간단하게"라는 말이 있으면 무조건 vanilla.** React, Expo, npm install 금지.

### 2. 프로젝트 생성 위치

- 항상 `web_init` 도구를 사용한다. `npx`, `npm create` 등을 직접 실행하지 않는다.
- `OUTPUT_DIR`은 항상 비워둔다 → `~/connect-ai-projects/`에 자동 생성됨.
- `kks` 폴더 안에 새 프로젝트를 만들지 않는다 (git 충돌).

### 3. CLI 명령어 규칙

- `npx`, `npm create`, `yarn create` 등 scaffold 명령에는 반드시 `--yes` 또는 `-y` 플래그를 붙인다.
- 대화형(interactive) 입력이 필요한 명령은 절대 그냥 실행하지 않는다.
- `web_init` 도구가 이미 이 규칙을 따르므로, 도구를 사용하면 자동 해결된다.
