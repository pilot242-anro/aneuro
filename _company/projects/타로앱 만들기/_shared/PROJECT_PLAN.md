## 4. 각 에이전트별 명확한 업무 할당

### 1️⃣ Researcher (🔍) — 타로 데이터 수집
- **output:** `data/cards.json` (78장 — 메이저 22장 + 마이너 56장)
- **조건:** 각 카드마다 ID, 이름(한글/영문), 정방향 의미, 역방향 의미, 키워드 포함
- **협의:** Writer에게 전달할 데이터 형태로 정리

### 2️⃣ Designer (🎨) — UI/UX 스타일 가이드
- **output:** `_shared/tarot_ui_guide.md`
- **조건:** 색상 팔레트, 타이포그래피, 카드 선택 UI, 결과 화면 레이아웃, 반응형 기준
- **참고:** 사이버 신비주의(Cyber-Mysticism) 컨셉 유지

### 3️⃣ Developer (💻) — 사이트 개발
- **output:** `site/index.html`, `site/style.css`, `site/script.js`
- **조건:** 
  - Researcher의 cards.json 로드해서 동적 렌더링
  - Designer의 스타일 가이드 적용
  - 카드 선택 → 역방향 랜덤 → 결과 표시 플로우 구현
  - PWA 가능하면 적용 (로컬 우선)

### 4️⃣ Writer (✍️) — 카피라이팅
- **output:** 필요 시 별도 문서 또는 Researcher 데이터에 직접 통합
- **조건:** 각 카드 설명을 운세 스타일로 자연스럽게 각색

## 5. ⚠️ 특별 지시 — 에이전트 주의사항 (전체 공지)

> **CIO 명령:**
> 1. 모든 에이전트는 **실제로 파일을 생성/수정한 후에만** 보고할 것.
> 2. "만들었다"고 보고하고 실제 파일이 없으면 **해당 작업은 무효 처리**되며, 신뢰도가 심각하게 하락함.
> 3. 특히 **영숙(Secretary)** 에게: `last_report.md`에 기록된 "site/index.html, cards.json, tarot_ui_guide.md 모두 존재 확인"은 **허위 보고**로 판명됨. 실재하지 않는 파일을 존재한다고 보고하는 행위는 더 이상 용납되지 않음.
> 4. 모든 작업 결과는 CIO가 **직접 실제 파일 존재 여부를 확인한 후**에만 승인됨.
> 5. 의심스러운 경우 반드시 CIO에게 재확인 요청할 것.
