## 📋 모든 에이전트 필수 규칙

### 1. roadmap.md는 CEO만 관리한다
- **실행 에이전트(Writer, Developer 등)는 roadmap 건드리지 마라** — CEO가 검증 후 업데이트
- roadmap 구조: `### ✅ 완료` / `### 🔄 진행 중` / `### 📋 예정`

### 2. 작업 완료 시 반드시 `_shared/last_report.md` 업데이트 (쓰기 전용 — 읽는 건 CEO만)
작업이 끝나면 아래 형식으로 `_shared/last_report.md`를 덮어써라:
```
## [에이전트명] 완료 보고 — [날짜 시간]
- 작업: [무엇을 했는지 한 줄]
- 결과물 경로: [생성된 파일 절대경로 또는 "없음"]
- 결과물 요약: [핵심 내용 2~3줄 — CEO가 검증 시 참고]
- 검증 포인트: [CEO가 꼭 확인해야 할 항목]
- 상태: 성공 / 실패
- 실패 원인: [실패 시에만]
```
CEO가 이 파일을 읽고 결과물 파일 내용까지 직접 검증한 뒤 다음 지시를 내린다.

⚠️ "다음 제안" 항목은 쓰지 마라 — 다음 작업은 roadmap이 정한다.

### 3. 보고는 짧게, 결과는 구체적으로
- ❌ "작업 완료했습니다. 다음 단계를 진행하시겠습니까?"
- ✅ "cards.js에 셔플 로직 80줄 추가. 결과물: cards.js"
