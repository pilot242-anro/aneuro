## 📋 모든 에이전트 필수 규칙

### 1. roadmap.md는 CEO만 관리한다
- **실행 에이전트(Writer, Developer 등)는 roadmap 건드리지 마라** — CEO가 검증 후 업데이트
- roadmap 구조: `## [프로젝트 이름]` → `### ✅ 완료` / `### 🔄 진행 중` / `### 📋 예정`
- **새 프로젝트 시작하면 새 `## 프로젝트명` 섹션 만들어라.** 기존 프로젝트 섹션 건드리지 마라.

### 2. 작업 완료 시 반드시 `_shared/last_report.md` 업데이트
작업이 끝나면 아래 형식으로 `_shared/last_report.md`를 덮어써라:
```
## [에이전트명] 완료 보고 — [날짜 시간]
- 작업: [무엇을 했는지]
- 결과물: [생성된 파일 절대경로 또는 "없음"]
- 상태: 성공 / 실패
- 실패 원인: [실패 시에만]
- 다음 제안: [자연스러운 다음 단계]
```
CEO가 이 파일을 읽고 검토한 뒤 다음 지시를 내린다.

### 3. 보고는 짧게, 결과는 구체적으로
- ❌ "작업 완료했습니다. 다음 단계를 진행하시겠습니까?"
- ✅ "카드 셔플 로직 80줄 추가 → cards.js. 다음: 78장 데이터 작성 시작."

### 3. 미루기 금지
- "이번 주 내로 환경 세팅" ❌
- "지금 바로 환경 세팅 시작" ✅
- 이 대화 안에서 끝낼 수 있는 건 무조건 이번 대화에서 끝낸다.
